### Контекст и цели
Я создаю собственную программу для автоматизации генерации видео с использованием HeyGen. Мне нужно определить оптимальный воркфлоу для моей задачи. 
Существуют два возможных воркфлоу, которые я хочу исследовать:
- **Воркфлоу А (Динамическая загрузка):** Пользователь загружает свое фото/аватар через мое приложение. Приложение отправляет это изображение в HeyGen, указывает текст для озвучки, выбирает определенный голос, и HeyGen генерирует видео.
- **Воркфлоу Б (Предзагруженный аватар):** Пользовательское фото уже загружено в HeyGen заранее. Мое приложение использует существующий `avatar_id`, передает текст и настройки голоса через API и генерирует видео.

Пожалуйста, проведи следующее исследование, используя свои возможности браузинга или внутреннюю базу знаний, начав с: [https://developers.heygen.com/docs/quick-start] и справочника API HeyGen (API Reference).

1. **Проверка осуществимости:** - Можно ли полностью автоматизировать Воркфлоу А (загрузка изображения через API и немедленная генерация из него говорящего видео) с использованием текущего API HeyGen? 
   - Если Воркфлоу А поддерживается не полностью или имеет строгие ограничения (например, фото-аватары в сравнении со стандартными аватарами), проверь Воркфлоу Б.

2. **Маппинг эндпоинтов API:**
